ReactNative-NavigatorIOS

ReactNative
  • NavigatorIOS

Demo screenshot

In this blog I will use a demo to show how ReactNative NavigatorIOS works.
The demo is written in ES6, you can check React docs to know more about it.
Below is the final app I made for this demo, you can get the code from my github repo. I use a very simple version of code to show the demo in this blog, be sure to have a look at the complete version on github if you are interested.

Why React Native?

As we know although React Native write apps with JS, all JS files will be compiled into native code at the end through its bridge.
I build iOS apps with both Swift and React Native, I love React Native because it’s more easy to debug.

DO NOT need to rebuild frequently

When I use Xcode and Swift to write native apps, it’s annoying that everytime even if I change a character, I have to rebuild the app to run, the rebuilding time can be long when code is much. But with React Native, I can just shake my real device or use CMD+R, it will reload the code.

Easy to inspect

React Native has a very powerful plugin in chrome, you can use that to inspect the elements on the screen just like inspecting a web page.

You can get more React Native debug information from React Native Debug Tool.

NavigatorIOS introdunction

React Native provide us many components thats native app has, and NavigatorIOS is one of them, it works just like the Navigator in native apps allowing you to add back-swipe functionality across the app.

How does NavigatorIOS works?

Speaking of NavigatorIOS, we must know two special properties: Route and Navigator

So let’s build an app while talking about Route and Navigator. I made some targets named: Requirement-1, Requirement-2,etc.

Requirement-1:

  1. Show a navigation scene when user enter the app.
  2. Show a right button on the header of the navigator.
  3. When press on the right button, navigate to FirstRightScene

A route is an object used to describe the page, if a page is rendered by a NavigatorIOS, we can get route in its prop.
